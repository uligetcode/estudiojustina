## 🎯 OBJETIVO DEL SITIO WEB

  

Crear un sitio web **estático de una sola página (landing page profesional)** con scroll vertical, que cumpla estos objetivos por orden de prioridad:

  

1. **Generar confianza** — que el visitante sienta que es un negocio serio y legítimo

2. **Comunicar claramente qué hacen** — sin ambigüedad, sin relleno

3. **Capturar contacto o consulta** — CTA claro hacia WhatsApp, teléfono o formulario

4. **Posicionar localmente** — que aparezca cuando alguien busca el rubro en esa ciudad

  

-----

  

## 🏗️ ESTRUCTURA OBLIGATORIA DEL SITIO

  

El sitio debe tener las siguientes secciones **en este orden lógico**:

  

### 1. `HEADER / NAV`

  

- Logo (texto estilizado si no hay imagen)

- Nombre del negocio

- Links de navegación internos (anclas hacia secciones)

- Botón de CTA principal visible: “Contactar por WhatsApp” o “Llamar ahora”

- El header debe ser sticky (fijo al hacer scroll)

  

### 2. `HERO / PORTADA`

  

- Headline principal impactante que comunique el beneficio central del negocio (NO el nombre, el BENEFICIO)

- Subheadline explicando qué hacen y dónde

- CTA principal (botón grande): “Pedir turno” / “Consultar ahora” / “Ver servicios” — según el rubro

- Fondo: si hay foto del local o imagen relevante, usarla. Si no, usar un gradiente o patrón coherente con los colores del negocio

- Debe transmitir confianza y profesionalismo en 3 segundos

  

### 3. `SOBRE NOSOTROS`

  

- Breve historia o descripción del negocio (2–4 párrafos cortos)

- Años de experiencia, equipo, valores, misión — lo que se tenga

- Si hay fotos del equipo o local, incluir aquí

- Humanizar el negocio: que el visitante sienta que hay personas reales detrás

  

### 4. `SERVICIOS / PRODUCTOS`

  

- Grid o tarjetas con cada servicio o producto principal

- Cada tarjeta: ícono o imagen representativa + nombre del servicio + descripción corta (1–2 líneas)

- Si se conocen precios, incluirlos. Si no, poner “Consultar precio”

- Si el rubro lo permite, destacar el servicio más popular o rentable con un badge: “Más solicitado” / “Recomendado”

  

### 5. `POR QUÉ ELEGIRNOS` (USP / Diferencial)

  

- 3 a 6 puntos fuertes del negocio en formato de íconos + texto corto

- Ejemplos genéricos que se pueden adaptar:

  - ✅ Atención personalizada

  - ✅ X años de experiencia

  - ✅ Profesionales certificados

  - ✅ Turnos flexibles

  - ✅ Materiales / insumos de primera calidad

  - ✅ Atención en el día

  

### 6. `TESTIMONIOS` (si hay datos o se pueden inferir)

  

- 2–3 testimonios de clientes (si se tienen datos reales, usarlos; si no, dejar placeholders editables con formato: “Nombre Apellido, [ciudad o descripción]”)

- Incluir estrellas de rating ★★★★★

- Dejar nota en el código: `<!-- Mantener sección vacía por ahora -->`

  

### 7. `GALERÍA / FOTOS` (opcional, solo si hay imágenes)

  

- Si hay fotos disponibles: grid de 3–6 imágenes del local, trabajo o productos

- Si no hay: omitir sección o usar placeholders con nota editable

  

### 8. `CONTACTO / CTA FINAL`

  

- Esta es la sección más importante en términos de conversión

- Debe incluir:

  - Botón grande de WhatsApp (con link directo: `https://wa.me/[número]?text=[mensaje+precodificado]`)

  - Número de teléfono clickeable (tel:)

  - Email clickeable (mailto:)

  - Horario de atención

  - Dirección física

  - Mapa de Google Maps embebido (usar iframe con la dirección)

  - Formulario de contacto simple (Nombre / Email o Teléfono / Mensaje / Botón Enviar) — aunque sea solo visual, funcional con mailto o formspree

  

### 9. `FOOTER`

  

- Nombre del negocio + año

- Links rápidos a redes sociales (con íconos SVG)

- Dirección y teléfono en una línea

- Texto legal mínimo: “Todos los derechos reservados”

## 🎨 DIRECTRICES DE DISEÑO

  

### Paleta de colores

  

- Detectar los colores del negocio a partir de la información provista

- Si no hay info de colores: inferir a partir del rubro:

  - Clínica / Salud → Azul oscuro o verde salud + blanco + gris claro

  - Belleza / Estética → Tonos tierra, nude, rosado oscuro o negro elegante

  - Alimentación / Restaurante → Cálidos: naranja, crema, marrón + verde

  - Ferretería / Industrial → Gris, amarillo, naranja oscuro

  - Educación → Azul medio + blanco + amarillo

  - Legal / Contable → Marino, gris oscuro, dorado

- Usar máximo 3 colores: primario, secundario y acento

- Definir las variables CSS al inicio del archivo

  

### Tipografía

  

- Usar Google Fonts — 2 fuentes máximo

- Una fuente para títulos (display): que tenga carácter propio y se alinee con el rubro

- Una fuente para cuerpo (body): legible, moderna, neutra

- Tamaños correctos en móvil y desktop

- **Prohibido**: Arial, Times New Roman, Comic Sans

  

### Layout y espaciado

  

- Generoso espaciado entre secciones (padding mínimo 80px top/bottom en desktop)

- Máximo ancho de contenido: 1200px centrado

- Grid responsive: 3 columnas en desktop → 2 en tablet → 1 en móvil

- Ningún elemento que se corte o desborde en pantallas pequeñas

  

### Micro-interacciones y animaciones

  

- Hover effects en botones (escala suave + cambio de color)

- Fade-in suave al hacer scroll en cada sección (Intersection Observer)

- Smooth scroll al hacer clic en la navegación

- Animación de entrada en el hero (fade + slide desde abajo)

- **No abusar**: máximo 1 animación por sección, que no distraiga

## ⚙️ REQUISITOS TÉCNICOS

  

### Stack

  

- **HTML5 + CSS3 + JavaScript vanilla** (preferido por portabilidad)

- O **React con Tailwind CSS** si la plataforma lo soporta mejor

- Un solo archivo HTML si es posible (CSS y JS embebidos o en `<style>` y `<script>`)

- Si se necesita separar: `index.html` + `styles.css` + `script.js`

  

### Performance

  

- Imágenes con `loading="lazy"`

- No depender de librerías externas pesadas (sin jQuery, sin Bootstrap)

- Íconos: usar SVG inline o Lucide Icons / Heroicons (CDN ligero)

- Fuentes de Google Fonts con `display=swap`

  

### SEO básico

  

- `<title>` con nombre del negocio + ciudad + rubro

- `<meta description>` descriptiva (155 caracteres)

- `<meta name="robots" content="index, follow">`

- Open Graph tags para compartir en redes:

  - `og:title`, `og:description`, `og:image`, `og:url`

- Headings jerárquicos correctos (H1 → H2 → H3)

- Alt text en todas las imágenes

- Schema.org markup básico para negocio local (JSON-LD):

  ```json

  {

    "@context": "https://schema.org",

    "@type": "LocalBusiness",

    "name": "...",

    "address": {...},

    "telephone": "...",

    "openingHours": "..."

  }

  ```

  

### Accesibilidad mínima

  

- Contraste de colores adecuado (WCAG AA)

- Atributos `aria-label` en botones sin texto visible

- Focus visible en elementos interactivos

- Navegable con teclado

  

### Mobile-first

  

- El diseño debe construirse pensando primero en móvil

- Menú hamburguesa en mobile (funcional con JS puro)

- Botón de WhatsApp flotante en mobile (esquina inferior derecha, siempre visible)

- Touch targets mínimo 44px

## 📝 INSTRUCCIONES ADICIONALES PARA LA IA

  

1. **No preguntes, ejecuta.** Con los datos provistos, toma decisiones de diseño y contenido. Si falta un dato, usa el más lógico para el rubro e incluye un comentario HTML indicando qué debe editarse: `<!-- COMPLETAR: agregar logo del negocio -->`

2. **Entrega el código completo.** No entregues fragmentos ni esquemas. El output debe ser código funcional listo para abrir en un navegador.

3. **Prioriza funcionalidad sobre perfección visual.** Un sitio que carga rápido, comunica claro y convierte es más valioso que uno bonito que no funciona en mobile.

4. **El copy (texto) importa tanto como el diseño.** No uses textos genéricos de relleno (“Lorem ipsum”). Escribe copy real y persuasivo basado en los datos del negocio. Cada headline debe resolver una pregunta o necesidad concreta del cliente potencial.

5. **Sé específico con la ciudad.** Mencionar el nombre de la ciudad múltiples veces ayuda al SEO local. Ejemplo: “La clínica de confianza en San Luis” > “Tu clínica de confianza”.

6. **El código debe ser limpio y comentado.** Usa comentarios `<!-- Sección: Servicios -->` para que el dueño del negocio (o yo) pueda editarlo fácilmente después.
